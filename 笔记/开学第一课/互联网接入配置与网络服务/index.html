互联网接入、配置与网络服务#

互联网接入#

拨号上网#

拨号上网是互联网发展早期的一种上网方式。用户通过电脑和调制解调器（Modem）拨打互联网服务提供商ISP的接入号码连接到互联网上。

调制解调器Modem在拨号上网过程中能将计算机产生的数字信号转成模拟信号，通过电话线传输到接收方，再通过接收方的调制解调器还原为数字信号送接收方的计算机。

拨号上网的网速一般只有几十Kbps到几Mbps，现在看来是非常慢的。

调制解调器Modem#

有线宽带上网#

早期采用拨号上网时，从家庭到ISP这最后一公里还靠普通电话线相连，但随着技术的发展，光纤入户越来越普遍，如果用户家庭到ISP之间装上了光纤，那么就能实现宽带上网。

这里所说的宽带，是指上网的传输媒体的带宽很宽。带宽又称为频带宽度，是指传输媒体在单位时间内可传输的数据量，一般以bps (bits per second，比特每秒)为单位，即每秒钟可传输多少个二进制位（bit）。带宽越宽，意味着在这个传输媒体上传输数据越快，所以我们通常将高速的传输媒体称为宽带。

光纤具有极高的带宽，且可以直接传输数字信号，所以通过光纤上网不再需要调制解调器进行“翻译”。

使用基于光纤的宽带上网，网速一般可达几百Mbps到几千Mbps以上。

家庭使用光纤宽带上网，通常需要有一个光纤接入的用户端设备，一般称为“宽带猫”或“光猫”。

因为以前拨号上网时将Modem称为“猫”，这个设备在外观上与Modem类似。

无线宽带上网#

WLAN#

WLAN（Wireless Local Area Network，无线局域网）是指以无线信道作传输媒介的计算机局域网络，即无线 局域网。

WLAN是在有线网的基础上发展起来的，它使上网的计算机具有可移动性，能快速、方便地解决有线方式不易实现的网络信道的连通问题。

目前WLAN常用频率为2.4G和5G的微波作为无线通信手段。

Wi-Fi#

Wi-Fi（Wireless Fidelity，无线保真）实际上是Wi-Fi联盟制造商的品牌认证，目的是改善基于IEEE802.11标准的无线网络产品之间的互通性，更好地实现无线连网。

一般符合IEEE802.11标准的产品和技术就被认为是Wi-Fi技术。

最简单的Wi-Fi上网就是设立一个无线路由器，在这个无线路由器的电波覆盖的有效范围都可以采用Wi-Fi连接方式进行联网。

IEEE802.11#

802.11标准是美国电气与电子工程师协会（IEEE）制定的一个WLAN标准系列，该标准定义了物理层和数据链路层中MAC子层的通信规范，允许计算机或网络设备按此标准来接入或组建无线局域网。

该标准类似IEEE802.3，只是IEEE802.3针对有线网，而IEEE802.11针对无线网。

802.11标准自1997年初次制定以来，随着技术的进步，已发展为一系列的WLAN标准，比较著名的有802.11a、802.11b和802.11ac等，各自工作在2.4GHz或5G开放频段。

无线路由器#

无线路由器是带有无线覆盖功能的路由器，主要用于无线上网，也可将宽带网络信号转发给周围的无线上网设备使用，如笔记本电脑、手机、平板电脑等。

无线路由器首先是一个无线网络的接入点，俗称“热点”，英文缩写为AP（Access Point，无线访问节点），它能够将有线网络信号转换为无线网络信号，并覆盖周边，让附近的无线上网设备能够接入到以无线AP为中心的无线局域网络。

无线路由器还具备宽带路由器的部分功能，具有网络地址转换功能，支持无线局域网的用户接入到对应ISP所在的互联网。

因此，无线路由器可看成是无线AP和宽带路由器合二为一的扩展型产品， 但它的路由功能相对于有线网络上的宽带路由器一般会弱很多。

无线网卡#

无线网卡的作用、功能类似有线网卡，所不同的只是其不采用有线连接方式，而是使用无线信号接入到无线局域网中。

网卡是一块实现了计算机网络的底层协议的硬件设备，通过网卡用户可以以有线或无线的方式接入网络。

每一个网卡都有一个被称为MAC地址的独一无二的48位串行号，它被写在卡上的一块ROM中，这也使得插入网卡的每一台计算机拥有一个唯一的MAC地址。

早期网卡以单独的物理硬件的形式存在，但目前新型电脑一般将网卡集成在主板上，大量的智能终端设备如手机等也将网卡的功能集成在主机电路板上。还有一些是外接USB接口的无线网卡，可插入到未配置无线网卡的台式计算机等设备中供其无线上网使用。

接入无线网络#

如果是接入学校、单位或公共场所的无线网络，只要在上网设备中选定要登录无线网的标识（Service Set Identifier，SSID），再输入密码完成认证后即可成功登录。

公共场所的开放无线网络，通常会以手机验证码验证的方式完成登录认证。登录成功后就可使用无线网络。

如果当前有线宽带和无线宽带均不可用，则还可通过移动电信运营商提供的蜂窝网络来上网。

蜂窝网络#

蜂窝网络（Cellular network）又称移动网络，主要是为手机等移动通信设备提供通信服务的网络。

  • 由于构成网络覆盖的各通信基地台的信号覆盖呈六边形，从而使整个网络像一个蜂窝而得名。

蜂窝网络的技术也经历了从模拟到数字，从1G到5G的发展历程。

  • 这里的“G”，是指“Generation”，是“代”的意思，“5G”就是第5代。

蜂窝网络经过多年的建设，已经成为移动通信的基础，覆盖范围广泛，通信安全可靠。随着5G技术的出现与成熟，蜂窝网的数据速率越来越快，可达到宽带的标准。

IP地址及其配置#

IP地址的概念#

互联网采用的协议是TCP/IP协议集。连在互联网上的计算机，如果与连在网上的其他计算机进行通信，就必须知道对方的地址，就像我们发邮件给朋友时必须知道朋友的邮箱地址一样。

互联网上的地址叫IP地址，每个联网的计算机都会被分配一个唯一的IP地址，通过这个地址我们可以区别互联网上每一台不同的计算机。

IP地址工作在TCP/IP模型的网络层（或称互联层）。目前主要有两种版本的IP地址：IPv4和IPv6。

IPv4#

当前在使用的绝大部分IP地址是IPv4（即IP地址版本4），它使用32位二进制（即4个字节）来表示一个地址。

为了便于阅读和记忆，地址采用“点分十进制”表示形式，即书写时将地址的4个字节每个字节写成十进制形式，字节与字节之间用英文的句点分隔。如202.120.80.1就是一个合法的IPv4地址。

一般IPv4地址的结构可分为两大部分：网络地址和主机地址。

IPv4地址的分类#
A类地址#

第一个字节作为网络地址，剩下的三个字节为主机地址。

对应较少的网络数量（126个），但每个网络可以容纳主机数很多。

A类地址的地址范围为1.0.0.1到127.255.255.254。

B类地址#

前二个字节为网络地址，后二个字节为主机地址。

B类地址适用于中等规模的网络，地址范围为128.0.0.1-191.255.255.254。

C类地址#

前三个字节为网络地址，剩下的一个字节为主机地址。

C类地址每个网络最多只能包含254台计算机，所以适用于小规模的局域网络。

C类地址的范围为192.0.0.1-223.255.255.254。

IPv6#

IPv6（Internet Protocol Version 6，互联网协议第6版），是互联网工程任务组（IETF）设计的用于替代IPv4的下一代IP协议。

IPv6的使用，不仅能解决网络地址资源数量的问题，而且也解决了多种接入设备连入互联网的障碍。

中国也极其重视IPv6的发展，2021年7月12日，中央网络安全和信息化委员会办公室、国家发展和改革委员会、工业和信息化部发布关于加快推进互联网协议第六版（IPv6）规模部署和应用工作的通知。

IPv6使用128位二进制（16个字节）来表示一个地址，大大扩展了地址空间。为未来智能终端接入互联网预留了足够的空间。

IPv6地址的表示采用“冒号分十六进制”格式，分为8组，每组16位，每组写成4个十六进制数，每组之间用冒号分隔。如：

  • FE80:0000:0000:0000:02AA:00FF:FE9A:4CA2
  • 可简写成：FE80::2AA:FF:FE9A:4CA2

IPv6简化了基于IP网络的实施、运营和管理，提高了路由的效率和可扩展性，能更有效地处理移动性并有更好的安全机制。

网络服务#

在计算机网络中，网络服务一般是指在网络上运行的、面向基础服务的软件模块。

互联网应用层有许多常见的网络服务，包括：

Web服务#

20世纪90年代初，WWW(World Wide Web，万维网)的发明，使Internet（因特网）迅速得到普及，并发展为连通世界各个角落的互联网。

WWW使用超文本标记语言HTML来组织信息，并可在网络上共享这些信息。接入互联网的用户只要使用浏览器和鼠标就可浏览世界各地的网页，查看新闻、下载文件、购买商品，这使得互联网的使用门槛大大降低，促使了互联网的爆炸性发展。

全球所有网站的网页组成了互联网的内容服务。这些网页又可分为静态网页与动态网页两类。

  • 静态网页是单纯用超文本标记语言HTML及客户端脚本语言（如JavaScript）编写，给所有访问者呈现的页面是相同的。
  • 动态网页是由程序动态生成的，对不同的访问者能呈现不同的页面。

互联网的精髓是通过互联网提供的内容服务。提供内容服务的是Web服务。Web服务的主要功能是提供网上信息浏览。

Web服务通常有3个要素：网页（HTML）、访问协议（HTTP）和浏览器。

HTML#

HTML（HyperText Markup Language，超文本标记语言）是一种独立于平台、固定格式的超文本标记语言

  • 用HTML制作的网页，浏览器能在任何平台上阅读它。这些网页又称为HTML文件，通常以.htm或.html为后缀。
  • HTML作为一种标记语言，其基本语法是在文本文档中嵌入一些标记（Tag），用以控制文本显示的外观。
  • HTML标记由一对尖括号"<“和”>" 包括，标记由英文字母组成，不区分大小写，本身并不在浏览器中显示，而用于提示浏览器如何处理包含在标记之间或在标记之后的文本。

HTTP#

HTTP（HyterText Transfer Protocol，超文本传输协议）是互联网上发布和接收超文本页面的规则和方法，是一个客户端和服务器端进行请求和应答的标准。

在HTTP协议中使用URL（Uniform Resource Locator，统一资源定位符）来定位服务器上的资源。

URL可看成是互联网上某一资源的地址。通常URL包括两部分：协议名和资源名。资源名又可由主机名、文件路径名、端口号和页内参照几部分组成。

HTTP协议工作原理#

客户端（用户）通过网页浏览器或其他工具（称为用户代理）发起一个到Web服务器（网页所在的网站）的请求，该请求的格式遵循HTTP协议或加了安全机制的HTTPS（超文本传输安全协议）协议。服务器在收到请求后，发回响应消息，该响应消息一般就是浏览器请求的网页文件。

路由服务#

路由器工作在网络层，通常用来连接多个网络或网段，如将本地局域网连入到互联网。

路由器最主要的功能是为网络层传输的IP分组选择一条合适的路由。

  • IP分组中会包含源地址和目的地址，从源到目的往往有很多条路径可以选择，路由器会读取每一个收到的分组中的目的地址，然后查寻保存在路由内部的路由表，根据路由表决定选择什么路径将分组传送出去。

路由器能够理解不同的协议，它可以分析和解读各种不同类型网络传来的分组的目的地址，再根据选定的路由算法把分组按最佳路径传送到目的计算机。

因为路由器最主要的功能是为经过路由器的分组寻找一条最佳传输路径，并将该分组无差错地传送到目的节点。因此，选择最佳路由的策略即路由算法是路由器的关键。

互联网（Internet）常用的路由算法有RIP（Routing Information Protocol，路由信息协议）、OSPF（Open Shortest Path First，开放最短通路优先）、BGP（Border Gateway Protocol，边界网关协议）等。

FTP服务#

FTP（File Transfer Protocol，文件传输协议）又称为文件传输服务，它可以让用户在两个连网的计算机之间传输文件。

在默认情况下，FTP协议使用TCP端口中的20和21两个端口，其中20用于传输数据，21用于传输控制信息。

DNS服务#

IP地址太长、太难记，DNS（Domain Name System，域名系统）的主要功能就是将一个IP地址关联到一组有意义的名字上去。

DNS定义了层次结构的名字空间，最上层节点称为顶级域名，第二层节点称为二级域名，依此类推，最下层的节点往往对应的是某台主机的域名。

某一个网站（或网上机构）的完整域名是由从该网站主机对应的底层域名到根节点对应的顶级域名之间所有域名连接而成，中间以点分隔。

有了DNS，用户就不需要死记硬背每个网站的IP地址，只要记住网站对应的域名就行了，域名与IP地址的转换由DNS服务来完成。

互联网域名与地址管理机构ICANN为不同的国家或地区设置了相应的顶级域名，通常由两个英文字母组成，例如：cn代表中国、uk代表英国、fr代表法国、jp代表日本。

美国因是互联网的起源国家，习惯上不用国家对应的顶级域名，常用代表机构类别的二级域名来作为顶级域名，如前面微软公司的完整域名（www.microsoft.com）中顶级域名就用代表公司或商业机构的com代替。

DNS是依靠DNS服务器（域名服务器）来实现IP地址与域名之间的 翻译的。

  • 首先，根据域名空间的层次结构将其按子树划分为不同的区域，每个区域必须有对应的域名服务器，存储该区域的域名信息。

  • 当用户需要查询某个域名对应的IP地址时，可直接向所在区域的域名服务器发查询请求，域名服务器在收到用户发出的请求后查询自身的资源记录集合，返回用户想要得到的最终答案，或者当自身的资源记录集合中查不到所需要的答案时，返回指向另外一个域名服务器（通常是其上一级的域名服务器）的指针，用户将继续向那个域名服务器发出查询请求。

DHCP服务#

DHCP（Dynamic Host Configuration Protocol，动态主机配置协议）对应的DHCP服务，通常应用在局域网环境中，主要作用是提供集中管理，使网络环境中的主机动态获得IP地址、子网掩码和DNS服务器地址等信息。

DHCP协议采用客户端/服务器模型，网络中应有一台DHCP服务器，由其完成IP地址的动态分配任务。申请IP地址的计算机先要向DHCP服务器发送申请地址的请求，DHCP服务器接收到该请求时，才会向申请者发送相关的地址配置等信息。

DHCP有三种机制分配IP地址：

  • 自动分配方式

    DHCP服务器为主机指定一个永久性的IP地址

  • 动态分配方式

    DHCP服务器给主机指定一个具有时间限制的IP地址

  • 手工分配方式

    客户端的IP地址是由网络管理员指定的，DHCP服务器只是将指定的IP地址告诉客户端主机。

三种地址分配方式中，只有动态分配可以重复使用客户端不再需要的地址。

Email服务#

Email服务又称为电子邮件服务，是指通过网络传送信件、单据、资料等电子信息的通信方法，是目前应用最广泛的互联网服务之一。

Email服务是根据传统的邮政服务模型建立起来的。当人们发送电子邮件时，这份邮件由邮件发送服务器发出，邮件发送服务器根据收件人的地址判断对方的邮件接收服务器而将这封信发送到该服务器上，收件人通过邮件代理软件从接收服务器获取邮件。

电子邮件服务涉及几个重要的邮件协议：

  • 邮件从邮件发送服务器传送到邮件接收服务器遵循SMTP（Simple Mail Transfer Protocol，简单邮件传输协议）协议。

  • 邮件代理从邮件接收服务器获取邮件有两种协议可用

    • POP3（Post Office Protocol – Version 3，邮局协议版本3）支持邮件代理从邮件接收服务器中取走邮件，在邮件代理所在的本地计算机中处理
    • IMAP4（Internet Mail Access Protocol 4，互联网消息访问协议版本4）则允许邮件仍留在邮件接收服务器中，邮件代理连网在线进行邮件处理。

VPN服务#

VPN（Virtual Private Network，虚拟专用网络），是指利用互联网等公共网络创建远程的计算机到局域网以及局域网和局域网之间的连接，从而构建一种跨越更大地理范围的局域网应用。

VPN属于远程访问技术，简单地说就是利用公用网络架设专用网络。

例如某公司员工出差到外地，他想访问企业内网的服务器资源，这种访问就属于远程访问。VPN让用户无论是在外地出差还是在家中办公，只要能上互联网就能利用VPN访问内网资源。
