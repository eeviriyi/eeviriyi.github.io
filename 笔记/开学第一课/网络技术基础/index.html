网络技术基础#

网络拓扑结构#

总线型结构/总线型网络#

总线型网络由一条总线连接若干个节点形成总线型结构。一般采用广播通信方式。作为总线的通信线路可以是双绞线、同轴电缆，也可以是光缆等。

总线型结构的特点：

  • 结构简单，可扩充，性能好
  • 网络的可靠性高，节点间响应速度快，共享资源能力强
  • 网络的成本低，设备投入量少，安装使用方便
  • 总线的性能和可靠性对网络有很大影响。

星型结构/星型网络#

星型网络是以中央节点为中心，把若干外围节点连接起来的结构。其中，中央节点对各外围节点间的通信和信息交换进行集中控制和管理。

星型结构的特点：

  • 建网容易，扩充性好，管理方便
  • 中央节点是整个系统的瓶颈，如果中央节点出现故障，则导 致整个网络瘫痪。

树型结构/树型网#

树型拓扑结构中，网络中的各节点（一般为计算机）形成了一个树型的层次化的结构。

树中低层计算机的功能和应用有关，一般都具有明确定义的和专业化很强的任务，如数据的采集和变换等，而高层的计算机还具备管理的功能，以便协调系统的工作，如数据处理、命令执行和综合处理等。

一般来说，树型结构的层次不宜过多，以免高层节点的管理负荷过重。

环型结构/环型网#

在环型拓扑结构构成的环型网中，节点通过点到点通信线路连接成闭合环路。数据将沿环中的一个方向逐站传送。

环型网结构简单，传输延时确定，但是环中每个节点与连接节点之间的通信线路都会成为网络可靠性的风险点，一旦出现故障就会影响整个网络的正常运行。

对于环型网，网络节点的加入、退出、环路的维护和管理都比较复杂。

网状型结构/网状型网#

网状型拓扑结构中，各节点通过传输线相互连接起来，并且任何一个节点都至少与其他两个节点相连。

网状型网络具有较高的可靠性，但其实现起来费用高、结构复杂、不易管理和维护。

广域网基本上采用网状型拓扑结构。

混合型拓扑结构/混合型网#

每一种拓扑结构都有自己的优缺点。一些大型网络通常都不是单一的网络拓扑结构，而是由多种拓扑结构混合而成，充分发挥各种拓扑结构的特长，这就是所谓的混合型拓扑结构。对应的网络即为混合型网。

互联网其实就是一个混合型网。

网络协议#

计算机网络是个非常复杂的系统。两台计算机之间要能够正常通信，需具备：

  • 数据传输的通路
  • 发送方获知接受方是否已经准备好接收数据
  • 接收方收到数据后需要知道如何识别这些数据（因为计算机实际收到的都是由0和1组成的二进制串）
  • 数据格式转换
  • 发送、接收同步
  • 差错或意外情况处理

这一切都需要通信双方高度协调，严格遵守事先约定的规则，这套规则就是网络协议。

网络的分层与封装#

一般网络协议都比较复杂。为了简化网络的设计，通常采用将网络“分层”的做法。

“分层”可将庞大而复杂的问题，转化为若干较小的、相对简单的局部问题，从而易于研究和处理。

网络分层后，各层之间是独立的，下一层通过层与层之间的接口为上一层提供服务。

由于每一层只实现一种相对独立的功能，复杂程度就大大下降了，带来灵活性好、易于实现和维护等好处。

将网络分层后，计算机网络的各层及其协议的集合，称为网络的体系结构。

网络的体系结构可看成是这一类计算机网络及其部件所应完成的功能的精确定义。

OSI模型#

OSI模型全称为开放系统互联基本参考模型（Open System Interconnection Reference Model），是国际标准化组织ISO在上世纪80年代制定的。

OSI模型有七个层次，概念清楚，但因其既复杂又不实用，没有在工业界得到呼应，也无相应的实物产品出现。该模型目前主要只在网络教学中使用。

TCP/IP模型#

TCP/IP模型是互联网（因特网）中采用的网络体系结构。

该模型源于实践，简单易用，在互联网上应用的成功也使 其在市场上得到了广泛的应用，成为事实上的工业标准。 目前几乎所有的网络设备都是兼容TCP/IP模型的。

TCP/IP模型实际只有应用层、传输层、网络层、网络接口层四层，网络接口层几乎为空，可以接纳不同类型的低层局域网络(含物理层和数据链路层)，典型的如以太网。

数据在网络各层之间的传递过程#

物理层#

物理层的任务是透明地传送比特流。其制定的规范有关传 输介质的特性标准。

物理层上所传的数据的单位就是二进制的比特（bit）。传递信息所利用的一些物理媒体，如双绞线、光缆等，并不包含在网络体系结构中。

物理层要考虑用多大的电压代表“1”或“0”，以及当发送端发出某个比特时，接收端如何来辨别这个比特是“1”还是“0”。

物理层还需确定连接电缆的插头应当有多少根引脚及这些引脚如何定义、如何连接。

数据链路层#

数据链路层的主要任务就是在两个相邻的计算机节点间无差错地传输以帧为单位的数据。

帧由若干个比特组成，是数据链路层传输的基本单位，包含数据和必要的控制信息，如同步信息、地址信息、差错控制信息、流量控制信息等。

数据链路层还会进行差错控制，如丢失重发、收发同步等。这样，数据链路层就将一条可能出错的实际链路，改造成一条无差错的数据链路供上面的网络层使用。

局域网是一个包含数据链路层和物理层两个层次的网络。因为局域网数据链路层的内容较多，通常分为两个子层

逻辑链路控制（Logical Link Control，简称LLC）子层

媒体访问控制（Medium Access Control，简称MAC）子层。

CSMA/CD协议#

CSMA/CD协议主要解决多个计算机站点争用共享传输信道（即总线）的问题。

该协议的工作原理可总结为四句话：“先听后发，冲突停发，边发边听，随机重发”。

MAC帧#

以太网的MAC帧，帧结构中通常包含目的地址、源地址、类型字段、帧数据及校验字段这些字段。

其中的源地址和目的地址，各占6个字节，称为MAC地址，也叫物理地址。这个地址通常对应的是网卡的地址，网络设备生产商在生产网卡时，会赋予这块网卡唯一的MAC地址。

网络层#

TCP/IP模型中的网络层通常又称为网际层（Internet Layer）或IP层。

网络层负责为互联网上的不同计算机提供通信服务。

这些计算机之间可能并不直接相连，两个计算机之间相互通信可能要经过许多个中间节点和链路，还可能经过多个通过路由器相连的通信子网。

在网络层，数据的传送单位是分组（或称为“包”）。

分组其实就是在数据链路层的帧前再加上一段网络层的控制信息，这段控制信息称为分组首部(或包头)。

网络层最主要的任务就是选择合适的路由，使发送方发送的分组沿合适的路径传输到接收方。同时，在网络中为通信双方确定一个唯一的地址，也是网络层的工作

传输层#

传输层的主要工作是负责端到端的通信。

这里的“端”，是指实际进行通信的计算机中的进程。进程是指正在运行的一段程序。

现在的计算机一般都支持多任务，一台机器上可能同时运行着多个应用。我们可以将一个应用看成一个进程（实际有的应用对应多个进程），发送方的进程是一个“端”，接收方的进程是另一个“端”，传输层就负责这两个“端”之间的数据传输。

TCP/IP模型的传输层主要有两种协议：

TCP协议（Transmission Control Protocol，传输控制协议）#

传输前需要先建立连接，然后在连接上传输，如发现差错需进行差错恢复。同时还有流量控制等机制。

这种方式是面向连接的服务，能够提供可靠的数据交付，但付出的代价是不灵活和效率低。

UDP协议（User Datagram Protocol，用户数据报协议）#

类似邮政系统的快递，每个分组自带目的地址，传输前不需要事先建立连接，而按当时的网络状况自行选择传输路径。

这种方式的数据是按“尽最大努力交付（best-effort delivery）”的原则来传输的，不能保证数据可靠的交付，但灵活、效率高。

应用层#

应用层是网络体系结构中的最高层，它直接为用户的应 用进程提供通信方面的服务。

TCP/IP模型的应用层，包含了大量互联网的应用协议。

如支持用户的网页访问（WWW、HTTP协议）、电子邮件传递（SMTP、POP协议）、文件传输（FTP协议）、远程登录（TELNET协议）等。

应用层的协议通常会使用下层协议提供的服务，如用HTTP协议访问网页时，就调用了传输层的TCP协议来建立数据通路。

网络传输介质#

导向性传输介质#

导向性传输介质是指信号在其内部会沿确定的方向传输的介质，通常指有线计算机网络中使用的传输介质，如同轴电缆、双绞线、光纤等。

同轴电缆#

同轴电缆是由同一轴心的内外两层导体组成，两层导体间有绝缘层，外层导体覆盖有网状屏蔽层（一般为金属材质），外面又包裹有绝缘保护套。

由于增加了屏蔽层，同轴电缆具有较强的抗干扰能力，能够保证数据在电缆中准确无误地传输较远的距离。

相比双绞线，同轴电缆价格较高，连网时需专用的连接头，导致组网成本高，且故障处理困难，所以现在已不太用于计算机的组网，而经常用于有线电视网络中。

双绞线#

双绞线（Twisted Pair）是一种最常使用的有线物理传输介质。它由两根绝缘的铜线互绞在一起而得名。

  • 将两根导线绞在一起的目的是减少来自其他导线中的信号干扰。通常由4对双绞线组成一根双绞线电缆。

相对于其他有线物理传输介质（同轴电缆和光纤）来说，双绞线价格便宜，也易于安装使用，但在传输距离、信道宽度和数据传输速度等方面均受到一定限制。

双绞线电缆可分成1至7类，类别越高，性能越好。

光纤#

光导纤维（Optical Fiber）简称光纤，是由能传导光波的石英玻璃等材料制造而成。

  • 光纤通信是利用光纤传递光脉冲来进行数据通信的（传输过程用到了光的全反射原理）。

  • 在传输介质上传输的数据一般是二进制形式，通常用光脉冲的“有”和“无”来表示传输数据中二进制的“1”和“0”。

光纤具有很高的数据传输速率、极宽的频带、低误码率和低延迟；并且光传输不受电磁干扰的影响，安全和保密性好；另外还具有重量轻、体积小、容易铺设的优点，多被用于长距离、大容量的主干传输网络中。

光缆则是由光纤、加强件和保护层等组成的线缆，按光缆中所含光纤的数量可分为单芯、双芯及多芯三类；按传输模式划分则可分为单模光纤和多模光纤两类。

非导向性传输介质#

无线电波#

无线电波一般是指频率在3kHz至300MHz之间的射频频段电磁波，是在自由空间（包括空气和真空）全向传输、覆盖范围较广的电磁波。

无线电波通信是利用地面的无线电波通过天空电离层的反射而到达接收端的一种远距离通信方式。

  • 由于大气中的电离层容易受到天气的影响，因此无线电波传输信号稳定性较差。

  • 但由于无线电波传输距离远，容易穿透建筑物，传输端和接收端不需要精准对准，因而被广泛用于室内和室外通信。

无线电波的传输特性与频率有关

  • 低频段无线电波能很容易绕过建筑物，但其能量随着传播距离的增大迅速衰减
  • 高频段无线电波趋于直线传播，容易被建筑物阻隔。
  • 实际使用无线电波传输数据时，通常采用低速传输，速率为几十至几百比特每秒。

常用无线电波传输电视和无线电广播信号。

微波#

微波一般是指频率在300MHz至300GHz之间的电磁波，比一般的无线电波频率高，通常也称为“超高频电磁波”。

微波是一种定向传输的电磁波，收发双方的天线必须相对应才能收发信息。

  • 微波通信有地面微波通信和卫星微波通信，通过中继站，可以实现远距离传播信号。

微波通信技术不仅用于电视、手机等无线通信设备的通信，也常用于局域网的无线通信。

  • 无线局域网（如Wi-Fi网络）的无线频率常为2.4G和5G；而手机使用的无线蜂窝网络，以4G网络为例，其频段在1.8G-2.6G之间。

红外线#

红外线一般是指频率在1012Hz至1014Hz之间的电磁波，频率比可见光低，是不可见光线。

红外线通信不受电磁干扰，可靠性很高，但传输距离只能在视线范围内，因此是一种近距离、低功耗、保密性强的通信方案，主要应用于小型移动设备之间近距离、“点对点”、直线式的无线数据传输。

电视机、空调机等家电使用的遥控器常用红外线传输信号。

激光#

激光一般是指频率在3.846 x 1014Hz至7.895 x 1014Hz之间的电磁波，是定向传输的电磁波。激光通信利用激光束调制成光脉冲用以传送数字信号，但它不能传送模拟信号。

激光通信也必须在可视范围内，它具有高度的方向性，所以保密性好，但缺点是传输距离有限，且会对周围环境造成影响。

蓝牙#

蓝牙技术是一种无线数据和语音通信开放的全球规范，最早由爱立信、IBM等5家公司在1998年联合推出，它使用全球通用的2.4GHz频段，支持设备短距离通信（一般10米内），是一种基于低成本的近距离无线连接技术。

蓝牙作为一种小范围的无线连接技术，能在设备间实现方便快捷、灵活安全、低成本、低功耗的数据通信和语音通信，通过它能在包括移动电话、PDA、无线耳机、笔记本电脑、相关外设等众多设备之间进行无线信息交换。

蓝牙的优点：

  • 适用设备多，无需电缆
  • 工作频段全球通用，适用于全球范围内用户
  • 蓝牙技术产品使用方便，利用蓝牙设备可以方便地搜索到另一个蓝牙设备并迅速建立起两个设备之间的联系
  • 安全性和抗干扰能力强等

蓝牙的缺点：

  • 传输距离较短（一般10米左右），数据传输速率也不高（通常低于2Mbps）。

网络设备#

不同计算机网络可以通过网络互连设备连接起来，以构成更大规模的网络，实现网络间的数据通信和资源共享。

网络互联设备的种类很多，分别工作在不同的网络协议层次上。

中继器#

中继器（repeater）用于完全相同的两类网络的互连，主要功能是通过对数据信号的重新发送或者转发来扩大信号的网络传输距离。

  • 一般电信号在通信介质上传输，距离越远损耗越大。中继器完成物理线路的连接，对衰减的信号进行放大和整形，使信号的波形和强度达到所要求的指标，然后再转发出去。
  • 中继器能延长网络的长度，常用于连接相同的局域网网段。

集线器#

集线器（Hub）是星型网络中普遍使用的高可靠性网络连接设备，采用IEEE802.3（以太网）标准的局域网大量使用双绞线和集线器来构建。

  • 每个集线器有多个端口，每个端口通过双绞线或光缆等传输介质与接入网络的计算机的网卡相连，形成一个星型的物理拓扑结构。
  • 当集线器接收到某个端口发来的数据比特时，就对信号进行再生、整形后简单地向所有其他端口转发。
  • 集线器的功能非常类似中继器，该设备也工作在物理层上。

网桥#

网桥（Bridge）也叫桥接器，是早期连接几个局域网网段的一种存储/转发设备。

  • 网桥通常有两个或多个端口，每个端口与一个网段相连。其主要功能是对收到的数据链路层的帧按目的地址进行转发。

网桥工作在数据链路层，可以使局域网的各网段既相互连接，又隔离了冲突，从而减轻了扩展的局域网的负载，这相当于过滤了每个网段的通信量。同时，使用网桥，可以连接不同类型和不同速率的局域网，扩展了网络的物理范围，也使网络的可靠性得到提高。

交换机#

这里的交换机（Switch）主要是指以太网交换机，它可看成是网桥的升级换代产品。

交换机也是工作在数据链路层的网络设备，其特点是可以为接入交换机的任意两个网络节点提供独享的电信号通路，这也是它与集线器的显著区别。

  • 与集线器将收到的信息向所有的端口进行转发不同，交换机会查看信息帧的目的地址，仅将信息帧向目的地址所在的网 段进行转发。

相比网桥，交换机转发延迟很小，转发性能更佳。

现代交换机通常还具有物理编址、错误校验、流量控制等功能。

  • 目前一些高档的交换机，还具备对虚拟局域网VLAN和链路汇聚的支持功能，甚至还具有路由和防火墙的功能。

由于交换机具有性价比高、高度灵活、相对简单和易于实现等特点，所以广受业界欢迎，其市场份额呈稳步上升态势。

路由器#

路由器（Router）是一种连接多个网络或网段的网络设备，工作在网络层上，又称为网关。

路由器最主要的功能是为经过路由器的分组寻找一条最佳传输路径，并将该分组无差错地传送到目的节点。

  • 路由器会读取每一个收到的分组中的目的地址，然后查寻保存在路由器内部的路由表，根据路由表决定选择什么路径将分组传送出去。路由表的内 容由所使用的路由协议决定。
  • 路由器能够理解不同的协议，它可以分析和解读各种不同类型网络传来的分组的目的地址，再根据选定的路由算法把分组按最佳路径传送到目的计算机。

路由器的优点：功能比网桥和交换机更为强大

  • 路由器可以连接不同类型的网络、解析分组中网络层对应的控制信息（含源地址和目的地址）、为分组选择最优路由等。
  • 有些路由器还具备一定的管理功能，能够监视数据传输，并向管理信息库报告统计数据；能够诊断内部或其他连接问题并触发报警信号。
  • 现代路由器还提供了对诸如VPN、防火墙、虚拟服务器、动态DNS等功能的支持。

路由器的缺点：数据处理速度相比交换机要慢很多

  • 为了满足一些高速网络中的连网要求，一些高端的交换机融合了部分路由功能，成为可以工作在网络第三层的高速交换设备。
