操作系统实例：Windows

操作系统实例：Windows Windows 文件系统 Windows中常见的文件系统有 FAT 和 NTFS 两种 FAT文件系统 一种较早期的文件系统，因该系统采用文件分配表（File Allocation Table, FAT）来管理文件，故称为FAT文件系统。 FAT是用来记录文件所在位置的表格，其中包含的是目录项，记录了每一个文件（或目录）的属性数据。 FAT文件系统使用“簇”作为最小的数据单元。“簇”是一组连续磁盘扇区的组合，所有文件（或目录）都存放在簇中。一个文件总是占用若干个整簇。 NTFS文件系统 NTFS（New Technology File System）是目前最常用的一种Windows标准文件系统。 NTFS大幅度地改善了原有FAT文件系统的性能，提供长文件名、数据保护和恢复，能通过目录和文件许可实现安全性，并支持跨越分区。 NTFS分区结构比FAT要复杂。NTFS分区主要由引导扇区、 主文件表、系统文件和文件存储区域等4个部分组成。 文件资源管理 文件资源管理器 应用程序管理 系统设置

操作系统概念

操作系统概念 操作系统的概念 操作系统的概念 操作系统，英文为Operating System，简称OS 操作系统位于计算机用户与计算机硬件之间，是管理计算机硬件的软件。它就像是一个管理硬件设备的管家，让这些硬件能够有效地协同工作，并为运行在上面的应用程序提供支持。 可以将操作系统看成是计算机用户和计算机硬件之间的中介，它提供一个承上启下的作用。其目的是提供环境，使用户能够快捷、高效地使用计算机。 不只是计算机上需要操作系统，几乎所有的智能设备上都可以看到操作系统的身影。我们平时使用的各种智能设备如手机、平板电脑，甚至一些电视、冰箱、洗衣机等智能家电，内部都包含了或复杂、或简单的操作系统。 操作系统的主要任务 处理如管理与配置内存、决定系统资源供需的优先次序、控制输入设备与输出设备、操作网络与管理文件系统等基本事务。 提供一个让用户与系统交互的操作界面。 不同视角看待操作系统 计算机用户关注的是操作系统为其提供的各项服务 程序开发人员关注的是操作系统为其提供的编程接口 操作系统的设计人员则关注系统中各式各样模块和单元的设计及它们之间的联系 操作系统与硬件 操作系统的主要功能是管理计算机硬件（Computer Hardware） 从系统架构的角度看，整个计算机系统可以粗分为四个层次，从下至上分别是： 硬件（Hardware） 操作系统 应用程序（Application Program） 用户（User） 计算机系统的组成 冯·诺依曼结构计算机，主要有五大部件组成： 运算器（Arithmetic Unit） 控制器（Controller） 存储器（Memory） 输入设备（Input Device） 输出设备（Output Device） 计算机的启动过程 当计算机开机或重启时，会运行保存在只读内存ROM中的一个叫做“引导程序”的初始程序。 引导程序先初始化系统的各个组件，从CPU寄存器、设备控制器到内存。 接着引导程序会定位操作系统内核并将其加载到内存中。 一旦操作系统内核加载到内存并运行，它就开始为系统和用户提供服务。 除了内核外，一些系统程序也在启动时加载到内存中运行并提供一些特定的服务，直到关机。 操作系统的功能和用户界面 进程管理 存储管理 设备管理 文件管理 用户接口 操作系统的类型 批处理系统 多道程序系统 分时系统 实时系统 通用操作系统 常见操作系统介绍 嵌入式操作系统 类Unix操作系统 Windows操作系统 手机操作系统 iOS Android

信息系统安全与信息社会责任

信息系统安全与信息社会责任 基本概念 通用计算机系统概述 计算机安全包括两个方面的内容：物理安全和逻辑安全 物理安全指系统设备及相关设施的安全性 逻辑安全是指信息的完整性、保密性和可用性，即信息的安全 物理损坏，以及人为的误操作而造成的损坏，可以通过硬件的双工，或数据冗余，或有效的备份措施进行防治 信息安全的基本术语 计算机信息系统 可信计算机 客体和主体 计算机信息的逻辑安全 识别（身份鉴别） 授权（访问控制） 安全审核（审计） 加密和签名 病毒防范 计算机信息系统安全保护等级划分 用户自主保护级 系统审计保护级 安全标记保护级 结构化保护级 访问验证保护级 常用的安全技术 加密技术 加密技术是通信双方按约定的法则进行信息特殊变换的一种保密技术。 将明文（Plaintext）为密文（Ciphertext）的过程称为加密（Encryption） 由密文恢复出原明文的过程，称为解密（Decryption） 加密技术是现代信息系统安全的基石。加密技术包括两个重要的组成部分：算法和秘钥。 算法是将普通的文本或者是可以理解的信息，与一串数字（秘钥）相结合，产生不可理解的密文的步骤 秘钥是用来对数据进行编码和解码的一种参数 对称式加密算法 收发双方使用相同秘钥的密码的加密算法被称为对称式加密 对称加密算法的优点是加密速度快、保密度高 非对称式加密算法 通信收发双方使用不同秘钥的密码，称为非对称式加密 非对称式加密算法的基本思想是：甲将公钥发送给乙，乙用公钥加密数据后，将加密后的数据再发送给甲，其他人即使截获了加密数据，因为没有密钥，也无法获取加密数据的真实内容 常见的加密算法 DES AES RSA Hash MD5 SHA-1 身份认证 身份验证是指通过一定的手段，完成对用户身份的确认。 大致可以分为以下三种: 基于共享密钥的身份验证 基于生物学特征的身份验证 基于公开密钥加密算法的身份验证 数字签名 数字签名（又称公钥数字签名）是只有信息的发送者才能产生的、别人无法伪造的一段数字串，这段数字串同时也是对信息的发送者发送信息真实性的一个有效证明 数字签名是一种类似写在纸上的普通的物理签名，但是使用了公钥加密领域的技术来实现的，用于鉴别数字信息的方法 一套数字签名通常定义两种互补的运算，一个用于签名，另一个用于验证 数字签名是非对称密钥加密技术与数字摘要技术的应用 访问控制技术 访问控制（Access Control）指系统对用户身份及其所属的预先定义的策略组限制其使用数据资源能力的手段。通常用于系统管理员控制用户对服务器、目录、文件等网络资源的访问 访问控制是系统保密性、完整性、可用性和合法使用性的重要基础，是网络安全防范和资源保护的关键策略之一，也是主体依据某些控制策略或权限对客体本身或其资源进行的不同授权访问 访问控制的主要目的是限制访问主体对客体的访问，从而保障数据资源在合法范围内得以有效使用和管理 为了达到上述目的，访问控制需要完成两个任务：识别和确认访问系统的用户、决定该用户可以对某一系统资源进行何种类型的访问 访问控制包括三个要素：主体、客体和控制策略 访问控制首先需要对用户身份的合法性进行验证，同时利用控制策略进行选用和管理工作 当用户身份和访问权限验证之后，还需要对越权操作进行监控 访问控制的内容包括认证、控制策略实现和安全审计 访问控制的主要功能包括：保证合法用户访问受权保护的网络资源，防止非法的主体进入受保护的网络资源，或防止合法用户对受保护的网络资源进行非授权的访问 防火墙技术 防火墙是一个由计算机硬件和软件组成的系统，部署于网络边界，是内部网络和外部网络之间的连接桥梁，同时对进出网络边界的数据进行保护，防止恶意入侵、恶意代码的传播等，保障内部网络数据的安全 防火墙技术是建立在网络技术和信息安全技术基础上的应用性安全技术，几乎所有的企业内部网络与外部网络（如因特网）相连接的边界设都会放置防火墙，防火墙能够起到安全过滤和安全隔离外网攻击、入侵等有害的网络安全信息和行为 防火墙按照不同的使用场景主要可以分成四类：...

信息系统的组成

通用计算机系统 通用计算机系统概述 现代信息系统的核心是计算机系统。自20世纪40年代诞生以来，计算技术保持高速发展，对人类社会产生了深刻广泛的影响，从而使得人类进入了信息社会。 现代通用计算机是基于冯·诺依曼结构的设计思想，称之为“冯·诺依曼”体系结构计算机系统。 计算机系统的两大组成部分 计算机硬件是构成计算机系统各功能部件的集合，是由电子、机械和光电元件组成的各种计算机部件和设备的总称，是计算机完成各项工作的物质基础。 计算机软件是指与计算机系统操作有关的各种程序以及任何与之相关的文档和数据的集合。计算机程序是用程序设计语言描述的、适合计算机执行的语句指令序列。 “冯·诺依曼”体系结构 计算机硬件由五个基本部分组成：运算器、控制器、存储器、输入设备和输出设备。 计算机内部采用二进制来表示程序和数据。 采用“存储程序”的方式，将程序和数据放入同一个存储器中（内存储器），计算机能够自动高速地从存储器中取出指令加以执行。 集成电路技术和摩尔定律 集成电路技术是制造微型机、小型机、大型机和巨型机的CPU的基本技术。 在1965年，芯片巨人英特尔公司的创始人戈登 · 摩尔提出了著名的摩尔定律：芯片上的晶体管数量每隔18个月左右就会翻一番。 计算机的主要功能 把需要的程序和数据送至计算机中。 必须具有长期记忆程序、数据、中间结果及最终运算结果的能力。 能够完成各种算术运算、逻辑运算和数据传送等数据加工处理的能力。 能够根据需要控制程序走向，并能根据指令控制机器的各部件协调操作。 能够按照要求将处理结果输出给用户。 计算机工作流程 取指令 指令译码 执行指令 访存取数 结果写回 计算机工作原理 首先读入程序和数据到主内存中。在控制器输入命令的控制下，通过输入设备把表示计算步骤的程序和计算中需要的原始数据读入到计算机的主内存中。 然后执行程序。在取指令作用下把程序指令逐条送入控制器。控制器对指令进行译码，并根据指令的操作要求向存储器和运算器发出存储、取数命令和运算命令，经过运算器计算并把结果存放在存储器内。 最后是输出结果。在控制器的取数和输出命令作用下，通过输出设备输出计算结果。 计算机系统硬件 五个基本部分 运算器 控制器 存储器 输入设备 输出设备 CPU 通常将运算器和控制器统称为中央处理器，即CPU（Central Processing Unit），它是整个计算机的核心部件，是计算机的“大脑”。CPU控制计算机的运算、处理、输入和输出等工作。 CPU主要由运算器和控制器两大部分组成 集成电路技术是制造微型机、小型机、大型机和巨型机的CPU的基本技术。 CPU的主要功能 指令控制：指令控制也称为程序的顺序控制，控制程序严格按照规定的顺序执行。 操作控制：为取出的指令产生一系列的控制信号(微指令)，分别送往相应的部件，从而控制这些部件按指令的要求进行工作。 时间控制：有些控制信号在时间上有严格的先后顺序，例如读取存储器的数据，只有当地址线信号稳定以后，才能通过数据线将所需的数据读出（否则读出的数据是不正确的数据），从而确保计算机有条不紊地工作。 数据加工：所谓数据加工，就是对数据进行算术运算和逻辑运算处理。 控制器（CU） 控制器CU（Controller Unit）由程序计数器、指令寄存器、指令译码器、时序产生器和操作控制器组成。 控制器是计算机的指挥系统，完成计算机的指挥工作。 控制器的基本功能是从内存读取指令和执行指令。 运算器（ALU） 运算器也称为算术逻辑单元ALU（Arithmetic Logic Unit）。 运算器由算术逻辑单元、累加寄存器、数据缓冲寄存器和状态条件寄存器组成。 运算器是数据加工处理部件，完成计算机的各种算术运算和逻辑运算。 CPU的架构 CPU按指令集可以主要分为如下两种类别：CISC（Complex Instruction Set Computers，复杂指令集计算机）和RISC（Reduced Instruction Set Computers，精简指令集计算机）。 CISC以Intel和AMD的X86 CPU为代表，而RISC以ARM和IBM Power为代表。 在计算机发展的数十年历史中，产生了各种架构的CPU。目前比较流行的架构主要包括以下4种：X86、ARM、RISC-V和MIPS。其它指令架构相对小众，只在专业的领域使用。 CPU的主要参数 CPU的主要参数有主频、外频、前端总线频率、倍频、高速缓存容量、所采用的制造工艺、功耗，以及CPU所采用的处理器架构、一系列的处理器计算技术等。...
